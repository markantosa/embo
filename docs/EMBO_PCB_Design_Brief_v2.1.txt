======================================================================
1. SYSTEM OVERVIEW
======================================================================
EMBO is a closed-loop embolic agent preparation device. The MCU board integrates three sensor subsystems (UAS, CV camera, StallGuard viscosity), two stepper motor actuators, two mechanical limit switches, and a touch-screen UI — coordinated by an ESP32-S3 communicating with a Raspberry Pi 5 via UART.

1.1 Why ESP32-S3-WROOM-1-N8
---------------------------
Module integrates chip, 8MB flash, 40MHz crystal, PCB antenna — no external MCU support components needed on your PCB
Built-in USB Serial/JTAG on GPIO19/20 — program via USB-C with no additional ICs
Bluetooth LE 5.0 on-chip — NimBLE library enables wireless serial debug stream to phone
36 GPIO broken out — 25 used in this design, 11 free for expansion
GPIO matrix: any peripheral routes to any GPIO — eliminates pin-conflict risk during layout
N8 chosen: 8MB flash sufficient for Arduino-ESP32 + BLE stack + OTA; no PSRAM keeps GPIO35–37 free for SPI bus
Avoid R8/R16 PSRAM variants — occupy GPIO35–37 internally. Avoid V suffix — GPIO47/48 at 1.8V on those.

Raspberry Pi 5 is powered by a separate external USB-C module. Do not connect power from the RPi UART connector to the RPi — it has its own isolated supply.

4. GPIO PIN ASSIGNMENTS
======================================================================
All GPIO numbers are ESP32-S3 logical GPIO numbers. The GPIO matrix allows routing any peripheral to any of these pins in firmware. All operate at 3.3V.
Strapping pins GPIO0, GPIO3, GPIO45, GPIO46: not driven by external circuits at power-up. GPIO1–GPIO18: brief ~60µs LOW glitch at power-up — motor EN pins on GPIO7/GPIO10 protected by 10kΩ pull-ups at TMC2209 side. GPIO26–GPIO32: reserved for internal SPI flash — never connect to external signals.
SPI BUS MODE NOTE — CRITICAL FOR FIRMWARE: AD9833 uses SPI Mode 2 (CPOL=1, CPHA=0 — clock idles HIGH). ILI9341/ST7789V and XPT2046 use SPI Mode 0 (CPOL=0, CPHA=0 — clock idles LOW). All share SPI2 on GPIO35–37. Firmware must switch SPI mode in software before each device transaction: Mode 0 for display/touch, Mode 2 for AD9833. Never mix in the same transaction.

5. UAS SIGNAL CHAIN
======================================================================
Generates 1MHz acoustic signal, transmits through syringe, measures received amplitude to infer particle attenuation. All on 3.3V rail, no virtual ground needed. Gain values verified against OPA2354 supply rail limits and dynamic range. Frequency sweep (0.5–3MHz) possible in firmware with same hardware.
OPA2354 is a DUAL op-amp (two channels per SOIC-8, labelled A and B only). Two packages are used: IC1 (Tx amp on ch-A, unused ch-B tied off) and IC2 (Rx stage 1 on ch-A, Rx stage 2 on ch-B). References to ch-C or ch-D are incorrect — the OPA2354 has no such channels.

---------------------------------------------------------------------------------------------------------------------

Supply           |  3.3V — both VDD pins                                                                             
SPI              |  GPIO38 (FSYNC), GPIO36 (SCLK), GPIO35 (SDATA) — write-only, no MISO. SPI Mode 2 (CPOL=1, CPHA=0).
RSET             |  6.8kΩ 1% 0402 from RSET to AGND. Produces IOUT_FS ≈ 3.02mA.                                      
COMP             |  10nF X7R 0402 to AGND — per AD9833 datasheet                                                     
IOUT load        |  200Ω to AGND → VOUT ≈ 604mVpp sine wave                                                          
Output coupling  |  100nF X7R 0402 from IOUT load to Tx amp non-inv input — blocks DC                                
Decoupling       |  100nF X7R 0402 within 2mm each VDD pin + 10µF X5R 0805 bulk

5.3 IC1 channel B — unused, must be tied off
--------------------------------------------
Unused op-amp channels must not be left with floating inputs — they will oscillate and inject noise into the 3.3V supply. Tie IC1 ch-B as a unity-gain follower.
IC1 pin 5 (non-inv input ch-B): connect to existing 1.65V mid-rail net (tap from any of the 10kΩ/10kΩ divider junctions elsewhere on the board)
IC1 pin 6 (inv input ch-B): connect directly to IC1 pin 7 (output ch-B) — unity-gain follower
IC1 pin 7 (output ch-B): leave unconnected to rest of circuit — just the tie-off loop above

5.10 ADC input — GPIO1 (ADC1_CH0)
---------------------------------
Voltage divider removed in v2.1. With total Rx gain = 100× and Vf(BAT54) = 0.2V, max envelope output ≈ 1.4V. ESP32-S3 ADC in 11dB attenuation mode accepts up to 3.1V. 1.4V is within range with 2.2× headroom. Resolution without divider: 3.1V / 4096 = 0.757mV/LSB — twice the resolution vs using the divider.

Component  |  Value           |  Function                                                                   
------------------------------------------------------------------------------------------------------------
D3         |  BAV99 SOT-23    |  Hard clamp at GPIO1 — overvoltage protection in case of gain miscalculation
C_adc      |  100nF X7R 0402  |  GPIO1 to AGND — noise filter at ADC input                                  

Connect envelope detector RC output (BAT54 cathode / R_env junction) directly to GPIO1 via D3 clamp
No voltage divider resistors — R_div_top and R_div_bot are NOT fitted (omit from BOM)
Trace from envelope node to GPIO1: under 10mm, no vias, routed over AGND pour
Firmware: configure ADC1_CH0 in ADC_ATTEN_DB_11 mode (11dB attenuation, input range 0–3.1V)
Baseline calibration: read GPIO1 with saline-only in syringe at startup, store as reference. Compute attenuation ratio on each subsequent read.

6. MOTOR DRIVERS — 2× TMC2209
======================================================================
Two TMC2209 ICs drive two NEMA 17 stepper motors. Shared half-duplex UART on GPIO4, addressed by MS1/MS2. StallGuard provides motor-load (viscosity proxy) data. SpreadCycle mode is hardware-enforced via SPREAD pin pull-up.

6.1 UART addressing
-------------------

Driver   |  Ref  |  MS1                            |  MS2               |  UART address
---------------------------------------------------------------------------------------
Motor 1  |  U5   |  GND (tie direct to GND plane)  |  GND (tie direct)  |  0           
Motor 2  |  U6   |  3.3V via 10kΩ pull-up          |  GND (tie direct)  |  1           

UART topology: UART1 TX and RX are both mapped to GPIO4 via the ESP32-S3 GPIO matrix in firmware (half-duplex mode). A single 1kΩ series resistor sits between GPIO4 and the junction node. Both PDN_UART pins of U5 and U6 connect to this junction node. This is the only UART resistor — do not add additional resistors at each IC.

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

VCC_IO               |  3.3V                   |  Logic supply for UART, STEP/DIR/EN signals                                                                                                                                   
VCP                  |  100nF X7R 0402 to GND  |  CRITICAL — charge pump output for high-side gate driver. Without this cap the gate driver will not switch and the motor will not run. Place within 2mm of VCP pin on each IC.

7. LIMIT SWITCHES
======================================================================
Two NO microswitches detect end-of-travel. Switch closure pulls GPIO to GND. Firmware immediately kills STEP PWM on trigger.

------------------------------------------------------------------------------------------------------------------------------------------------------------
GPIO14  |  LIMIT_M1  |  J8 — JST-XH 2.54mm 2-pin  |  Pin 1: GND. Pin 2: GPIO14. NO microswitch between pins. 100nF from GPIO14 to GND for hardware debounce.
GPIO15  |  LIMIT_M2  |  J9 — JST-XH 2.54mm 2-pin  |  Pin 1: GND. Pin 2: GPIO15. NO microswitch between pins. 100nF from GPIO15 to GND for hardware debounce.

Enable internal pull-ups in firmware (GPIO_PULLUP_ONLY). Software debounce (5ms filter) additionally required in ISR. 8. TFT TOUCH-SCREEN INTERFACE
======================================================================
Module: Shopee SPI TFT Serial Port Module 5V/3.3V PCB Adapter with Micro SD slot — ILI9341 or ST7789V controller, XPT2046 resistive touch. The module has an onboard LDO (for 5V input) and level shifters. For this design, the 3.3V pin on the module is used directly, bypassing the onboard LDO.

8.1 Power — use 3.3V input pin
------------------------------
Connect module VCC (or the dedicated 3.3V pin on the PCB adapter) to the 3.3V rail — NOT the 5V VCC pin
At 3.3V, the onboard level shifters on the module are transparent — SPI signals from ESP32-S3 pass through at 3.3V with no translation needed
Do NOT connect the 5V VCC pin of the module — the 3.3V input is sufficient and avoids powering the onboard LDO unnecessarily
Confirm correct pin on the specific Shopee module: the PCB adapter typically labels both a 5V and a 3V3 pad — use 3V3

-----------------------------------------------------------------

-----------------------------------------------------------------------------------------------------------------------------------------------------------

CRITICAL FIRMWARE NOTE — SPI clock speed: the ILI9341/ST7789V display accepts up to 40MHz SPI clock. The XPT2046 touch controller maximum SPI clock is 2MHz. Since they share SPI2, firmware must explicitly reduce the SPI clock speed before each touch read and restore to 40MHz before the next display write. Missing this step will produce random or zero touch readings.
CRITICAL FIRMWARE NOTE — SPI mode: AD9833 requires SPI Mode 2 (CPOL=1, CPHA=0). ILI9341/ST7789V and XPT2046 require SPI Mode 0 (CPOL=0, CPHA=0). Firmware must switch SPI mode in software between AD9833 and display/touch transactions. This is purely a firmware responsibility — the hardware supports both modes on the same bus.
CONTROLLER VARIANT: The Shopee module description lists both ILI9341 and ST7789V. Confirm which chip is on the specific unit received — the initialization command sequences are different. Both are SPI Mode 0 and 240×320 resolution. Firmware team should detect or hard-code the correct driver before exhibition.
TOUCH CALIBRATION: XPT2046 returns raw 12-bit ADC values (0–4095), not pixel coordinates. A first-run calibration routine (display three cross-hair targets, user taps each) generates three mapping coefficients. Store in ESP32-S3 NVS (Non-Volatile Storage) in flash — no external EEPROM needed. Calibration persists across power cycles.
SD CARD SLOT: the module includes a micro SD slot sharing the SPI bus (GPIO35/36/37). If the SD card is not used, ensure the SD CS pin is pulled HIGH (connect to 3.3V via 10kΩ if exposed). A floating SD CS can cause the SD card to interfere with display SPI traffic.

10. PCB LAYOUT RULES
======================================================================

10.1 Layer stack
----------------
2-layer FR4, 1oz copper, ENIG
Top: all SMD components + signal routing
Bottom: ground plane pour + power routing where needed
UAS analog signals on top layer only — no vias in UAS signal paths

10.2 ESP32 antenna keepout — mandatory
--------------------------------------
No copper, traces, vias, or components within 15mm of PCB antenna on ALL layers
Remove ground plane in keepout zone on bottom layer — include this in KiCad board setup
Place module at board edge with antenna pointing outward

10.3 Ground strategy
--------------------
Single GND net — no separate AGND/DGND copper pours
Route UAS analog traces directly over the uninterrupted ground pour
All component GND pins, thermal pads, TMC2209 exposed pad → bottom ground plane via vias

10.4 LM2596 power section
-------------------------
Catch diode 1N5822: within 5mm of SW pin, same layer, oriented to minimise diode-inductor-cap loop area — this is the switching loop and minimising it reduces EMI on the UAS analog chain
10Ω + 1nF snubber from SW pin to GND: place within 2mm of SW pin, minimises ringing
Input capacitors: directly at VIN pin
Output capacitors: directly at output node, before bleed resistor junction
Inductor: between SW pin and output caps — avoid running inductor current-carrying traces over analog signal area

10.5 AMS1117 thermal
--------------------
SOT-223 tab (pin 4, output net): must connect to a top-layer copper pour of at least 100mm² area
Connect top-layer pour to bottom GND plane via minimum 4 thermal vias (0.3mm drill)
Without this thermal connection the LDO will overheat at 850mW dissipation

10.6 UAS signal chain
---------------------
IC1 (Tx amp) and IC2 (Rx amp): ≥15mm physical separation
J2 SMA → R_in → D1 BAV99 → IC2 ch-A non-inv input: under 20mm, no vias, ground guard trace each side
IC2 ch-B output → D2 BAT54 → envelope RC: within 5mm of IC2
Envelope RC output → D3 BAV99 → GPIO1: under 10mm, no vias
All OPA2354 decoupling caps within 2mm of V+ pins
AD9833 RSET and COMP caps within 3mm of respective pins

10.7 TMC2209 motor drivers
--------------------------
VCP cap (100nF) directly at VCP pin of each IC — within 2mm, not shared
VM decoupling (100µF + 100nF) directly at each VM pin — separate per IC
Current sense resistors: Kelvin connections — sense traces routed separately from coil power traces
All 8 motor coil traces: ≥0.5mm width
UART junction node: short, symmetric, equal-length traces to each PDN_UART
Keep motor output traces away from SPI and UAS analog traces — opposite end of board if possible
SPREAD pull-up resistor: within 5mm of SPREAD pin on each IC
