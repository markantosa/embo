1. SYSTEM OVERVIEW
======================================================================
EMBO is a closed-loop embolic agent preparation device. The MCU board integrates three sensor subsystems (UAS, CV camera, StallGuard viscosity), two stepper motor actuators, two mechanical limit switches, and a UI breakout board connected via IDC ribbon cable.

1.1 Why ESP32-S3-WROOM-1-N8
---------------------------
Module: chip, 8MB flash, 40MHz crystal, PCB antenna — no external MCU support components needed
Built-in USB Serial/JTAG on GPIO19/20 — no bridge IC needed
Bluetooth LE 5.0 on-chip — NimBLE for wireless debug
36 GPIO broken out — 28 used in this design, 8 free
N8 variant: no PSRAM — GPIO35–37 free for SPI bus

6. MOTOR DRIVERS — 2× BIGTREETECH TMC2209 V1.X PLUG-IN MODULES
======================================================================
U5/U6 changed from TMC2209-LA QFN-28 to BTT TMC2209 V1.x plug-in modules in 2×8 sockets. Eliminates QFN-28 thermal pad as single point of failure; field-replaceable.
FUNCTIONAL SENSOR INPUT: StallGuard SG_RESULT feeds the PID mixing-stop algorithm — affects the science, not just motion. Depends on sense resistor, coil current, and SpreadCycle mode.

6.1 Module specifications
-------------------------
Fixed onboard sense resistor: 110mΩ 1%, confirmed via BTT docs, consistent V1.1–V1.3. Same class as original RS1–RS4 — scaling math unaffected.
Current set via onboard trimmer pot (VREF) by default — main source of run-to-run variability vs original design.
DIAG/INDEX broken out but NOT used — viscosity sensing via UART-polled SG_RESULT.

6.2 UART addressing
-------------------

Driver   |  Ref  |  MS1            |  MS2  |  Address
-----------------------------------------------------
Motor 1  |  U5   |  GND            |  0  
Motor 2  |  U6   |  3.3V via 10kΩ  |  GND  |  1      


6.3 PDN_UART pad — confirmed location
-------------------------------------
VERIFIED: all BTT TMC2209 module variants (V1.1, V1.2, V1.3) route the active UART signal out of the 4th pin down (left column), labeled PDN, by factory default. The 5th pin (adjacent) is a spare PDN pad, not a duplicate — route only the 4th pin.

Component  |  Value  |  Connection          
--------------------------------------------
R_UART     |  1kΩ    |  GPIO4 → PDN junction
R_PDN_UP   |  10kΩ   |  PDN junction → 3.3V 


6.4 SpreadCycle — firmware-enforced
-----------------------------------
CRITICAL CHANGE: this module has NO SPREAD pin. SPREAD pull-up resistors REMOVED from BOM entirely.
SpreadCycle required for StallGuard4. Firmware MUST force via UART at every boot:
Write GCONF.en_spreadcycle=1 over UART to both U5 and U6
Read back DRVSTATUS/GCONF to confirm the write succeeded
HARDWARE VERIFICATION GATE: manual load test on real module, confirm SG_RESULT responds before trusting data for PID tuning

6.5 CLK pin — correction from v2.1–v2.4
---------------------------------------
CORRECTION: prior versions incorrectly stated CLK floats for internal oscillator. Per datasheet, CLK must tie to GND.
CLK (6th pin down, left column): tie to GND via 0Ω resistor or direct trace — 2 total

6.6 VDD pin — unverified
------------------------
NOT CONFIRMED: 3.3V assumed safe for module VDD but not verified against specific module schematic. Confirm before powering on.

10. PCB LAYOUT RULES
======================================================================

10.1 Layer stack
----------------
2-layer FR4 ENIG. UAS analog top layer only, no vias.

10.2 Antenna keepout
--------------------
No copper within 15mm of ESP32 antenna on ALL layers.

10.3 Power section
------------------
Q1/D7 within 3mm. SS34A within 5mm of SW pin. AMS1117 thermal vias mandatory.

10.4 UAS chain
--------------
IC1/IC2 ≥15mm separation. C_adc within 2mm of GPIO1.

10.5 TMC2209 module sockets (CHANGED)
-------------------------------------
SK1, SK2: 2×8 0.1" sockets, hand-solderable — no reflow dependency for these positions.
Hot-plate reflow requirement previously specific to QFN-28 no longer applies to U5/U6. Confirm separately for AD9833/OPA2354.
VM bulk caps remain on main board near each socket.
Confirm enclosure clearance for module+socket height with mechanical team.

10.6 IDC ribbon (J9) — unchanged
--------------------------------
100nF cap from TOUCH_IRQ trace near J9.

10.7 USB-C routing
------------------
D+/D− as single nets across A/B pads. CC resistors within 5mm.
